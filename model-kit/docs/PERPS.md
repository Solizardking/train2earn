# Perps Tool Lane

The model kit includes the Solana perps tool layer from `ai-training/perps/`.
This does not copy generated bytecode or caches. It references the source files
and writes reviewable manifests into generated data lanes.

## Included Source

| File | Role |
| --- | --- |
| `perps/functions.py` | OpenAI/Hermes function-call tools for SOL price, token price, Phoenix markets, funding, orderbook, positions, balances, Jupiter quotes, paper trades, market overview, trader history, SOL send simulation, and risk scoring. |
| `perps/functioncall.py` | Hermes-3 function-calling loop over the perps tools. |
| `perps/nvidia_perps.py` | NVIDIA signal-discovery handoff writer for observer/paper perps runs. |
| `perps/prompter.py` | Standard and GOAP prompt builders for perps tools. |
| `perps/schema.py` | Pydantic schemas for function calls, trade orders, risk assessments, portfolios, and market signals. |
| `perps/README.md` | Source perps toolkit documentation and safety contract. |

`perps/__pycache__/` is generated Python bytecode and is intentionally excluded.

## Commands

Run from the repo root:

```bash
ai-training/model-kit/bin/clawd-model-kit perps tools
ai-training/model-kit/bin/clawd-model-kit perps manifest
ai-training/model-kit/bin/clawd-model-kit perps handoff --market SOL --mode observer
```

Write the model-kit manifest:

```bash
ai-training/model-kit/bin/clawd-model-kit perps tools --write --json
```

Default output:

```text
ai-training/data/model_kit/perps_tool_manifest.json
```

Run one paper signal tick:

```bash
export RPC_URL=https://api.mainnet-beta.solana.com
ai-training/model-kit/bin/clawd-model-kit perps handoff \
  --market SOL \
  --mode paper \
  --tick
```

Run the Hermes perps function-calling agent:

```bash
export HF_TOKEN=hf_...
ai-training/model-kit/bin/clawd-model-kit perps agent \
  --query "Assess the risk of long SOL-PERP 500 USDC at 2x"
```

Local adapter mode:

```bash
ai-training/model-kit/bin/clawd-model-kit perps agent \
  --local \
  --adapter solanaclawd/solana-nvidia-trading-factory-8b-lora \
  --goap \
  --query "Build a paper-only SOL-PERP plan with risk checks"
```

## Safety Contract

- Default modes are `observer` and `paper`.
- `LIVE_TRADING=true` is refused by the model-kit wrapper unless
  `--allow-live-env --yes` is passed after review.
- Private keys, wallet passwords, OAuth files, and API tokens must stay in the
  shell or secret manager, never in prompts, configs, manifests, or Hub uploads.
- Tool output is model context, not execution authority.
- Live Phoenix/Vulcan execution belongs outside the model-kit generator and
  requires preflight, margin review, position review, market review, and an
  explicit operator decision.
